# BACKUP STRATEGY SUMMARY
# Date: April 7, 2025
# Author: Donald Tanner
# ----------------------------------------------------------------

=== Detailed Backup Strategy & Data Flow ===

1. Borg Backup System:
   ├─ Local Repository: Daily backups at 22:00
   │  └─ Backs up: ~/Documents, ~/Projects
   ├─ Remote Storage: Backblaze B2 Buckets
   │  └─ Same data as local repository
   └─ Monitoring: Weekly backup status checks (cron)

2. Duplicity Backup System:
   ├─ Cloud Storage: Backblaze B2 Buckets
   ├─ Backs up: ~/.config, ~/.gnupg, ~/.bash*, ~/Documents, ~/Notebooks
   ├─ Schedule:
   │  ├─ Daily Incremental: 02:00
   │  ├─ Monthly Full: 03:00 (1st of month)
   │  └─ Monthly Cleanup: 04:30 (1st of month)
   └─ Monitoring: Daily backup status checks

3. Dropbox System (Manual):
   ├─ Purpose: Quick access & file sharing
   ├─ Management: Custom shell functions & aliases
   └─ No automated backups (intentional)

4. DeJa Dup (Supplementary):
   ├─ Storage: Google Drive
   ├─ GUI-based backup solution
   └─ Backs up: Selected personal data

5. Vorta (Borg GUI Frontend):
   ├─ Remote Storage: Backblaze B2 Buckets (same as Borg CLI)
   ├─ Local Repositories: Various manually controlled local repos
   ├─ Management: Manual scheduling and verification
   └─ Purpose: GUI-based supplement to CLI Borg backups

=== Cross-System Integration ===

* Backblaze B2:
  ├─ Primary cloud storage platform
  ├─ Used by: Borg CLI, Vorta (Borg GUI) & Duplicity
  └─ Separate buckets for each system

* Cloud Storage Distribution:
  ├─ Backblaze B2: Primary backup storage
  ├─ Google Drive: DeJa Dup backups
  └─ Dropbox: Manual file management

=== Monitoring Integration ===

Cron Jobs Monitor:
├─ Borg: Daily backup verification
├─ Duplicity: Daily backup checks
└─ Weekly comprehensive status check
